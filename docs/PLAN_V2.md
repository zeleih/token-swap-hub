# 第二期功能扩展方案 — Dashboard 增强

基于第一期的核心平台，本期将新增 7 项功能以优化用户体验和社交互动。

## 一、需求清单与数据库变更

### 1. 平台 API URL 展示与复制
- 在 Dashboard 增加 **Base URL** 卡片。
- URL 从环境变量 `PLATFORM_URL` 读取（默认 `http://localhost:3000/api/v1`），部署时改为正式域名即可。

### 2. 用户名注册（替代强制邮箱）
- 注册表单将"邮箱"改为"用户名"，推荐但不强制使用邮箱格式。

#### 数据库变更
```diff
 model User {
-  email         String    @unique
+  username      String    @unique  // 用户自定义用户名（推荐但不强制邮箱）
+  displayName   String?             // 用户昵称（可后续更改）
```

### 3. 用户信息编辑
- Dashboard 顶栏增加用户信息按钮，点击弹出个人信息编辑面板。
- 可编辑项：**昵称 (displayName)**、**是否上排行榜 (showOnLeaderboard)**。

### 4. 排行榜（贡献榜 + 消费榜）
- Dashboard 顶栏增加"排行榜"按钮，点击弹出 Modal 弹窗展示两个 Tab：
  - **贡献榜**：按提供的 Token 被消耗的总量倒排
  - **消费榜**：按用户累计消费的 tokens 倒排
- 支持 **日榜/周榜/月榜/年榜/总榜** 5 个时间段。
- 仅展示 `showOnLeaderboard = true` 的用户。

### 5. OAuth 方式添加 Token
- 在"添加 Token"表单中增加 **"通过 OAuth 授权"** 选项卡。
- 目前为 UI 占位和框架预留，不实际对接。

### 6. Token 使用上限与定向开放
- 每个 Token 可设置：
  - **usageLimit**：调用上限（null 表示无限制）
  - **allowedUsers**：允许使用该 Token 的用户名列表（逗号分隔，空表示对所有人开放）
- **定向开放的 Token 双方均不计费**（消费者不扣点，提供者也不赚点）。
- 代理路由在选取 Token 时过滤：跳过已达上限的、以及不在白名单中的。

### 7. 信用点数赠送
- Dashboard 增加"赠送点数"按钮，弹出表单输入：**对方用户名** + **赠送数量**。
- 需要手动输入 CONFIRM 进行二次确认。
- 使用事务 (Transaction) 确保扣减与增加的原子性。

---

## 二、Token 选取规则（重要）

代理路由在选择 Token 时，**只允许以下三类 Token**：
1. ✅ 消费者**自己提供的** Token
2. ✅ **管理员注入的**公共 Token（`isAdminSupply = true`）
3. ✅ 被**定向开放**给该消费者的 Token（白名单中包含该用户名）

> ⚠️ **不会使用其他用户的 Token**。如果没有可用 Token，系统直接返回 503 错误。

## 三、其他改进

### 左侧菜单栏
- 新增可折叠的左侧导航菜单，包含"控制台"和"工具箱"两个入口。
- 工具箱页面为占位，方便后续拓展额度管理、多平台监控等功能。

### 主题切换
- 顶栏增加亮/暗模式切换按钮。

### 公告栏
- Dashboard 底部增加公告区域，展示平台最新公告和使用说明。
