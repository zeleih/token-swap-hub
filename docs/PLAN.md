# 闲置 Token 抽水蓄能与共享平台方案设计

这套系统旨在为您和朋友们提供一个互助共享 Token 的统一入口。用户可以将闲置的 AI Token (官方 URL) 提交到平台，系统调用这些 Token 响应其他用户的请求。平台基于"信用点数"运作，实现 Token 的共享、流转和消费。整个平台界面支持中英双语一键切换。

## 1. 业务核心逻辑 (Core Business Logic)
- **提供者 (Provider)**: 在平台填入自己的官方 Token（比如 OpenAI api key）。系统会在后台验证其有效性。当其他用户通过定向开放使用你的 Token 时，可以赚取信用点数。
- **消费者 (Consumer)**: 在个人界面获取唯一的分配凭证（平台 URL + 专属 Key）。将这套凭证填入支持自定义接口的 AI 客户端（如 Chatbox、Raycast、Cursor 等）。调用时系统会优先使用你自己的 Token、管理员公共 Token 或定向开放给你的 Token。
- **积点流转机制 (Credit System)**:
  - 核心思想：每位用户管理和消费自己的 Token，通过定向开放功能与朋友互助共享。
  - 管理员可注入公共 Token 作为新用户体验基础，定向开放的 Token 双方不计费。

## 2. 核心功能模块 (Core Modules)

### 2.1 用户与认证系统
- **邀请制注册 (Invite-Only)**: 管理员能在后台生成邀请码，非邀请码用户无法注册。
- **登录鉴权**: 使用自定义用户名/密码登录，结合 JWT 认证。
- **个人中心**: 显示当前信用点数余额，近期调用消耗记录，并提供快捷复制"平台接口 URL 与专属 Key"的按键。

### 2.2 Token 池与管理系统
- **添加与监控**: 用户添加 Token 时，平台自动检测连通性和余额。对连续请求失败（如 401 封禁、429 频率超限）的 Token 实施熔断或冻结。
- **状态提示**: 实时标记 Token 状态（活跃、冻结、无效），并统计它的历史服务次数。
- **使用上限与定向开放**: 每个 Token 可设置使用上限和白名单，仅允许指定用户使用。

### 2.3 代理分发与计费路由 (Proxy & Routing)
- **API 兼容层**: 拦截发往 `/v1/chat/completions` 等官方格式的请求。在请求头上验证访问者的"平台专属 Key"。
- **Token 选取规则**: 只从以下三类 Token 中选取：① 消费者自己提供的 Token ② 管理员注入的公共 Token ③ 定向开放给该消费者的 Token。不会使用其他用户的 Token。如果没有可用 Token，直接返回错误。
- **拦截器计费**: 在 HTTP Stream 响应完或请求结束后，按 OpenAI 返回的精确 token 用量计费。定向开放的 Token 双方不计费。

### 2.4 中英文多语言 (i18n)
- 界面顶栏/侧边栏提供明显的 `中/En` 按钮，通过无刷新方案切换 UI 文字。所有的提示、表单和系统邮件均为双语。

## 2.5 计费与点数管理机制 (已确认)
- **精准计费模型**: 根据 OpenAI 官方计算的精确 Tokens 用量进行扣减，例如 1 token 对应消耗/获得 1 信用点。
- **预体验点数与管理员资金池**:
  - 新进邀请的系统圈子朋友默认获得一定数额的"启动体验点数"。
  - 系统为管理员预留单独的 Token 注入接口（Admin Portal）。管理员可挂载批量购买的便宜 Token 作为公共资金流（水库），以此支撑新用户免费体验。

## 3. UI 交互与视觉设计
- 采用深色/亮色模式可切换，玻璃拟态（Glassmorphism）组件，构建充满极简现代感与科技感的前端界面。
- **UI 组件**: Tailwind CSS + shadcn/ui + 左侧可折叠菜单栏。

## 4. 技术栈推荐
- **核心框架**: Next.js App Router (可无缝支持全栈、中间件代理转发)
- **UI 组件**: Tailwind CSS + shadcn/ui
- **数据库**: PrismaORM + SQLite (体积小、轻量、无需配置外部服务，非常适合小范围朋友圈使用。未来可直接迁移到 PostgreSQL)
- **i18n**: `next-intl` 实现多语言

## 4. 验证计划 (Verification Plan)
1. **多语言与路由验证**: 启动 Next.js 本地开发服务，验证中英文点击无缝切换。
2. **邀请制注册链路测试**: 人工模拟生成一条邀请码，并用新浏览器窗口尝试进行有效/无效邀请码注册拦截测试。
3. **代理转发与扣费断言**: 
   - 使用模拟（Mock）的 Token 发起一次对话请求。
   - 检查消费者积分是否正确扣除，提供者积分是否对应增长。
   - 检测路由代理是否能透传流式输出（Streaming）而不中断。
